## 📐 이미지 사양 요구사항

### 파일 크기
- **최대**: 500KB 이하 권장
- **최소**: 100KB 이상 (품질 유지)

### 해상도
- **DPI**: 72-96 (웹 최적화)
- **컬러 모드**: RGB

### 포맷
- **PNG**: 투명 배경 필요 시, 텍스트 많은 경우
- **JPG**: 사진, 일반 이미지 (압축 효율 좋음)

## 🎨 디자인 가이드라인

### 후기 이미지 (review-*.png/jpg)

**1. 카카오톡 스크린샷**
- 상대방 이름은 익명 처리 또는 별명
- 민감한 개인정보 블러 처리
- 대화 흐름이 자연스럽게 보이도록
- 핵심 문장에 형광펜 강조 추천

**2. 이메일 후기**
- 발신자/수신자 정보는 익명 처리
- 핵심 문장 하이라이트
- 날짜/시간은 유지 (신뢰도)

**3. 텍스트 이미지**
- 배경: 흰색 또는 연한 색상
- 폰트: Pretendard, Noto Sans KR
- 크기: 본문 14-16px, 강조 18-20px
- 여백: 충분한 패딩 (20-30px)

## 🚀 업로드 방법

### 1. 로컬 개발 환경
```bash
# images/ 폴더에 파일 직접 복사
프레임-설계자/
└── images/
    ├── review-4.png
    ├── review-1.png
    ├── review-1.jpg
    ├── review-7.png
    ├── review-3.jpg
    ├── book-cover.jpg
    └── coach-profile.jpg
```

### 2. FTP/SFTP
- FileZilla, Cyberduck 등 사용
- `/images/` 폴더에 업로드

### 3. Git 저장소
```bash
git add images/review-4.png
git add images/review-1.png
git add images/review-1.jpg
git add images/review-7.png
git add images/review-3.jpg
git add images/book-cover.jpg
git add images/coach-profile.jpg
git commit -m "Add review and profile images"
git push
```

## ✅ 업로드 후 확인사항

### 1. 파일 존재 확인
- [ ] 모든 파일이 `images/` 폴더에 있는가?
- [ ] 파일명이 정확한가?
- [ ] 확장자가 소문자인가?

### 2. 브라우저 확인
- [ ] 후기 섹션에 4개 이미지 모두 표시되는가?
- [ ] 프리미엄 혜택 증명 이미지가 보이는가?
- [ ] 책 표지와 프로필 사진이 보이는가?

### 3. 모바일 확인
- [ ] 모바일에서 이미지가 잘 보이는가?
- [ ] 로딩 속도가 적절한가?
- [ ] 반응형으로 크기가 조정되는가?

### 4. 품질 확인
- [ ] 이미지가 선명한가?
- [ ] 텍스트를 읽을 수 있는가?
- [ ] 압축으로 인한 왜곡이 없는가?

## 💡 팁 & 주의사항

### ✅ DO (해야 할 것)
- 개인정보 보호 (익명 처리)
- 적절한 압축 (로딩 속도)
- 일관된 스타일 (통일감)
- 저작권 확인 (허가받은 이미지)

### ❌ DON'T (하지 말아야 할 것)
- 너무 큰 파일 크기 (1MB 이상)
- 흐릿한 저품질 이미지
- 개인정보 노출
- 저작권 침해 이미지
