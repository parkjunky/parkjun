프레임 설계자 랜딩 페이지의 모바일 UX를 전면 개편하여 단일 열 레이아웃, 햄버거 메뉴, 터치 최적화, 이미지 lazy loading 등을 구현했습니다.

**구현 내용:**

### 2. 단일 열 레이아웃 (모바일) ✅

**변경 사항:**

모든 2단 그리드를 단일 열(1fr)로 변경:

| 섹션 | 이전 | 이후 |
|------|------|------|
| 문제 카드 | 2단 | 1단 |
| 프레임워크 스텝 | 2단 | 1단 |
| 통계 그리드 | 2x2 | 1단 |
| 책 비교 | 2단 | 1단 |
| 후기 그리드 | 2x2 | 1단 |
| 이유 카드 | 2단 | 1단 |
| 주요 특징 | 2단 | 1단 |
| 주문 비교 | 2단 | 1단 |
| 가격 인포그래픽 | 2-3단 | 1단 |

**효과:**
- 더 넓은 콘텐츠 영역 (좌우 여백 축소)
- 스크롤하며 읽기 편한 레이아웃

**적용 범위:**
- 768px 이하: 1단 레이아웃
- 480px 이하: 더 컴팩트한 1단 레이아웃
- 769px 이상: 기존 레이아웃 유지 (변경 없음)

### 5. 이미지 최적화 ✅

**Lazy Loading 적용:**
- 모든 이미지에 `loading="lazy"` 속성 추가
- 총 10개 이미지에 적용:
  - book-cover.jpg
  - coach-profile.jpg
  - review-7.png
  - review-3.jpg
  - main-review-1.jpg ~ 4.jpg
  - more-review-1.jpg ~ 2.jpg

**구현 방식:**
1. HTML: `<img loading="lazy">`
2. CSS: 로딩 전 opacity 0 → 로딩 후 opacity 1
3. JS: Intersection Observer로 로딩 완료 감지

**효과:**
- 초기 페이지 로딩 속도 향상
- 데이터 사용량 절약
- 부드러운 이미지 페이드인 효과

**반응형 처리:**
```css
img {
    max-width: 100%;
    height: auto;
    display: block;
}
```

### 7. 4포인트 그리드 시스템 ✅

**적용된 값:**
- 8px, 12px, 16px, 20px, 24px, 28px, 32px, 40px

**변경 사항:**
```css
/* 이전 */
margin-bottom: 12px;  /* 유지 */
padding: 14px 20px;   /* → 16px 20px */
gap: 10px;            /* → 12px */

/* 이후 (모두 4의 배수) */
margin-bottom: 16px;
padding: 16px 20px;
gap: 12px;
```

**효과:**
- 디자인 일관성 향상
- 개발/유지보수 편의성
- 시각적 리듬감
