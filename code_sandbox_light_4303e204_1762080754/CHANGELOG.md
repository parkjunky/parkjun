# 변경 이력 (Changelog)

## [1.5.0] - 2025-01-02

### 📱 모바일 UX 전면 개편 - 완전한 단일 열 레이아웃

#### 핵심 변경사항

**1. 햄버거 메뉴 내비게이션**
- 모바일(768px 이하)에서 햄버거 메뉴 아이콘 추가
- 터치하기 쉬운 44x44px 버튼 크기
- 부드러운 슬라이드 애니메이션
- 메뉴 외부 클릭 시 자동 닫힘
- 메뉴 링크 클릭 시 자동 닫힘

**2. 단일 열 레이아웃 (768px 이하)**
- 모든 2단 그리드를 1단으로 변경:
  - 문제 카드 (`.problem-cards`)
  - 프레임워크 스텝 (`.framework-steps`)
  - 통계 그리드 (`.stats-grid`)
  - 책 비교 그리드 (`.book-comparison-grid`)
  - 후기 그리드
  - 이유 카드 (`.reason-cards`)
  - 주요 특징 (`.key-features`)
  - 주문 비교 (`.order-comparison`)
  - 가격 비교 인포그래픽
  - 프리미엄 코칭 비교
- 더 넓은 콘텐츠 영역으로 가독성 대폭 향상

**3. 아코디언 시스템**
- 긴 설명 섹션에 접기/펼치기 기능 추가
- 기존 feature-box, toc-box 아코디언 유지
- 새로운 범용 아코디언 컴포넌트 추가
- 부드러운 애니메이션 효과

**4. 터치 최적화**
- 모든 버튼: 최소 48px 높이
- 메인 CTA 버튼: 56px 높이
- 네비게이션 링크: 44px 터치 영역
- FAQ 질문: 48px 터치 영역
- 하단 고정 바 버튼: 56px 높이

**5. 이미지 최적화**
- 모든 이미지에 `loading="lazy"` 속성 추가 (10개 이미지)
- `max-width: 100%` 반응형 처리
- Intersection Observer API로 lazy loading 구현
- 로딩 완료 시 부드러운 페이드인 효과

**6. 타이포그래피 개선**
- 본문 텍스트: 16px (모바일)
- 줄간격: 1.5 (최적 가독성)
- 제목 크기 반응형 조정

**7. 4포인트 그리드 시스템**
- 모든 패딩을 4의 배수로 정리:
  - 12px, 16px, 20px, 24px, 28px, 32px
- 마진도 동일하게 적용
- 시각적 일관성 향상

#### 기술적 개선
- JavaScript 햄버거 메뉴 토글 로직 추가
- Intersection Observer로 이미지 lazy loading
- 범용 아코디언 이벤트 핸들러 추가
- CSS 애니메이션 최적화

#### 영향
- ✅ **모바일 가독성**: 단일 열로 콘텐츠 집중도 향상
- ✅ **사용성**: 터치하기 쉬운 버튼 크기
- ✅ **로딩 속도**: Lazy loading으로 초기 로딩 시간 단축
- ✅ **접근성**: 햄버거 메뉴에 aria-label 추가
- ✅ **디자인 일관성**: 4포인트 그리드 시스템
- ✅ **데스크톱**: 769px 이상에서는 기존 레이아웃 100% 유지

## [1.4.1] - 2025-01-02

### 📱 모바일 최적화 개선 - 더 넓은 콘텐츠 영역 확보

#### 핵심 변경사항
- **컨테이너 여백 추가 축소**
  - 768px 이하: `padding: 16px` → `12px` (25% 감소)
  - 480px 이하: `padding: 12px` → `10px` (추가 17% 감소)
  - 결과: 모바일 화면에서 좌우 24-40px 더 넓은 콘텐츠 영역 확보

- **2단 레이아웃 완벽 유지 및 최적화**
  - 주요 특징 영역 (`.key-features`): `flex-direction: column` → `grid-template-columns: repeat(2, 1fr)` 변경
  - 주문 비교 섹션 (`.order-comparison`): `grid-template-columns: 1fr` → `repeat(2, 1fr)` 변경
  - 무료 전자책/프레임 설계자 비교: 2단 유지 확인
  - 문제 카드, 프레임워크 스텝, 통계, 후기, 이유 카드: 모두 2단 유지

#### 영향
- ✅ 모바일 가독성 향상: 더 넓은 콘텐츠 영역으로 텍스트 읽기 편안
- ✅ 정보 밀도 최적화: 2단 레이아웃 유지로 스크롤 길이 최소화
- ✅ 시각적 균형: 좌우 여백 감소로 전체적인 디자인 균형 개선
- ✅ 데스크톱 레이아웃: 769px 이상에서는 기존 레이아웃 100% 유지

## 버전 관리 정책

### 버전 번호 체계
- **Major.Minor.Patch** (예: 1.2.1)
- **Major**: 대규모 변경, 호환성 깨짐
- **Minor**: 새로운 기능 추가
- **Patch**: 버그 수정, 내용 업데이트

### 변경 타입
- ✨ **기능 추가** (Features)
- 🐛 **버그 수정** (Bug Fixes)
- 📝 **내용 수정** (Content Updates)
- 🎨 **디자인 개선** (Design)
- ⚡ **성능 개선** (Performance)
- 🔧 **설정 변경** (Configuration)
